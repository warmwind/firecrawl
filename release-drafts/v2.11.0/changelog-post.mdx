## v2.11.0 is live

Firecrawl v2.11.0 lands PII redaction directly on scrape, makes the Monitor API a lot smarter with LLM-judged goals and JSON-mode diffs, splits the `searchZDR` forced mode into explicit ZDR and anonymous variants, and migrates self-host deployments from the Supabase SDK to Drizzle on native Postgres. We also added a `WWW-Authenticate` header for agent discovery, raised the PDF size cap, added V1-compatible aliases on the V2 SDKs, and patched a fresh batch of CVEs.

### Highlights

- **`pii` format and `redactPII`** — Set `redactPII: true` on `/v2/scrape`, `/v2/batch/scrape`, `/v2/crawl`, `/v2/parse`, or `/v2/extract` and the worker runs the scraped markdown through fire-privacy and returns the redacted text as `document.markdown`. Add the `pii` format to also get spans, counts, and per-entity diagnostics. Costs +4 credits per scrape (plus +4 per extra PDF page). Exposed across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Monitor goals with LLM judge** — Set `goal` and `judgeEnabled` on a monitor and every changed page is classified as meaningful vs noise, sorted into summary emails with the unified diff rendered inline. Typed in JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Monitor JSON diffs and snapshots** — Monitors using `{type: "json"}` or `{type: "changeTracking", modes: ["json"]}` now diff structured fields against the previous run and persist a `snapshot.json` of current values, surfaced through `GET /v2/monitor/:id/checks/:checkId`.
- **Mixed-mode change tracking** — Requesting both `["json", "git-diff"]` modes now runs both diffs and reports `changed` if either side changes, with the markdown unified diff attached as a sidecar on the JSON artifact.
- **Monitor email confirmation** — External monitor recipients now confirm opt-in before they receive alerts and get a one-click unsubscribe link in every summary email; team members are auto-confirmed.
- **AM/PM monitor schedules** — Daily schedule strings like `daily at 9am` or `daily at 5:30pm` are accepted and converted to the right UTC cron.
- **Agent discovery on `401`** — Unauthorized responses now include `WWW-Authenticate: Bearer realm="firecrawl"` so MCP and agent clients can discover the credential scheme programmatically.
- **`forced-zdr` and `forced-anon` search modes** — `searchZDR: "forced"` has been split into `"forced-zdr"` and `"forced-anon"` to disambiguate ZDR enterprise routing from anonymous routing; `"forced"` continues to work as a deprecated alias.
- **Self-host Drizzle migration** — The API now runs on Drizzle ORM against native Postgres. Self-host deployments switch from `SUPABASE_*` env vars to `DATABASE_URL`, optional `DATABASE_REPLICA_URL`, and `INDEX_DATABASE_URL`; Stripe auto-recharge is removed in favor of Autumn-driven credit top-ups.
- **PDF size cap raised to 50 MB** — PDF download and scrape payloads can now be up to 50 MB (up from 30 MB).
- **V1 aliases on V2 SDKs** — Python and JS V2 clients now expose deprecated `scrape_url` / `scrapeUrl`, `crawl_url` / `crawlUrl`, and friends so V1-trained callers keep working without rewriting code.
- **JS SDK string API key** — `new Firecrawl("fc-key")` now works alongside the object form, with whitespace-only keys rejected up front.
- **CVE patches** — Resolved `axios` (GHSA-35jp-ww65-95wh and friends) in the JS SDK, the Rust SDK's `openssl` (CVE-2026-41676), and pnpm-overrode `ws`, `brace-expansion`, `qs`, `uuid`, and `js-cookie` across the API, JS SDK, Playwright service, test-suite, and ingestion UI.
